## 运行验证 + 修复两处 bug（P0+P1 收尾）

用户要求「能不能运行一下」。本沙箱无法启动微信开发者工具，故对后端做端到端实跑验证。

### 验证结果
- 后端全链路跑通：登录→系统题→加用户题→优先抽用户题→批量导入→列表→删。
- 发现并修复 **2 个真实 bug**：
  1. 后端 `server/src/index.js` 缺 `PUT /api/questions/:id` 路由（只写了 DELETE），且 `readBody` 只解析 POST/DELETE，PUT 拿到空 body。已补 PUT 路由 + 把 PUT 加入 CORS 方法与 body 解析。改怒气值/题目文字均验证通过，不存在的题返回 404。
  2. 前端 `mp/services/question.js` 缺 `update()` 方法，「手动改怒气值」无调用入口。已补 `update(id, patch)` 用 PUT（题库页内联编辑 UI 属 P2，按之前约定需与用户确认细节再动）。
- 重启时遇 `EADDRINUSE`：旧后台进程未释放 3000 端口（pkill 在 git bash 不生效）。改用 PowerShell `Get-NetTCPConnection -LocalPort 3000` 定位 PID 后 `Stop-Process -Force` 强杀，再重启成功。

### 环境注意
- git bash 下 `taskkill //F` 斜杠被转义失败，且 Bash 内调用 PowerShell 被安全策略拦截 → 端口占用一律用 PowerShell 工具处理。
- git bash 路径用 `D:/Code/...`（大写盘符+正斜杠）才能让 node --check 正确解析；`/d/...` 会被拼成 `d:\d\...` 报错。
