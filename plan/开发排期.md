# 真心话抽卡对战小程序 · 开发排期

> 目标：微信小程序端「真心话抽卡对战」游戏。
> 四大核心功能：①单人抽卡 ②用户系统 ③题库管理（含 AI 评分）④西部对战。
> 排期原则：**地基先行 → 单机核心 → 社交对战 → 迭代打磨**。对战模式重依赖题库、用户、实时同步，故置于单机三件套之后。

---

## P0 基础设施（必做 · 前置）
- 项目脚手架：app.json 路由/tabBar、全局样式 app.wxss、sitemap.json、project.config.json
- 统一请求层 `utils/request.js`：Promise 化、鉴权头、401 自动刷新重试
- 微信登录：wx.login → code → 后端换 openid/session；头像昵称走最新规则（`button open-type="chooseAvatar"` + `input type="nickname"`）；隐私协议弹窗（`wx.requirePrivacyAuthorize`）
- 数据模型：
  - 题目表 `questions`（id, content, anger 0-10, ai_reason, source[system/user], owner_openid）
  - 用户表 `users`（openid, nick, avatar, level, exp, hp_base）
  - 房间表 `rooms`（room_id, players[], status, turn, hp[], bullet, current_question）
- 题库种子：内置一批系统题（普通弹药来源）
- 环境配置：dev / staging / prod 切换（request BASE_URL + 云环境）

## P1 单人抽卡模式（必做 · 核心）
- 老虎机滚动动画抽取题目（多列滚动 → 减速停止）
- 正反翻转动画（rotateY 180°）展示题目内容
- 读取题库（系统题 + 用户自建题混合）
- 分享能力：`onShareAppMessage` + `onShareTimeline`

## P2 题库管理（必做 · 核心）
- 手动添加：输入内容 → 调 AI 评怒气值(0-10) + 评分依据 → 用户可改数值
- 剪贴板批量导入：解析多行/分隔符，逐条 AI 评分
- 题目列表：编辑、删除、怒气值展示
- AI 评分封装层：可替换 provider（混元/OpenAI/规则）

## P3 西部对战模式（核心 · 最重）
- 房间：创建 + 微信分享邀请（onShareAppMessage 带 `roomId`）；对手点卡片进入房间
- 入场状态：1 人 → 弹窗提示「邀请对手」；2 人 → 弹窗「是否开战」
- 先手：掷骰子 或 邀请者优先（待确认）
- 弹药：高级弹（用户题，伤害=怒气值）/ 普通弹（系统题，伤害=怒气值×0.8）
- 回合流程：装弹 → 开枪 → **子弹飞行中途停滞展开题目** → 受击方回答 → 发射方选倍率（不满意×1.2 / 普通×1.0 / 较好×0.5 / 超好×0）→ 受击 / 阵亡
- 像素风牛仔对枪动画（Canvas / 帧动画 / Skyline）
- 血量：L1=20，每级 +10；血量变动带入场/飘字动画
- 结算页：伤害值、回答评价、综合评级 S/A/B/C/D → 经验值 → 升级

## P4 迭代（后续）
- 音效 / 震动反馈
- 排行榜、好友对战记录
- AI 自动生成题目
- 更多场景皮肤（除西部外）
- 3+ 人房间
- 订阅消息提醒对手回合（`wx.requestSubscribeMessage`）

## 关键风险 / 待确认（首轮）
1. **后端选型**：云开发 vs 自建服务 → 决定对战实时同步方案
2. **AI 评分 API**：混元 / OpenAI / 规则 / 先留接口
3. **首版范围**：是否含完整对战（影响交付周期与评审复杂度）
4. **美术资源**：代码绘制像素风 / 用户提供素材 / emoji 占位

## 合规注意
- 所有 wx.request 域名需在 MP 后台配置白名单且 HTTPS
- 隐私协议 + 用户授权前置（登录、剪贴板、位置等）
- 用户生成内容需过 `msgSecCheck` / `imgSecCheck`
- 主包 < 2MB，对战/营销模块走分包
