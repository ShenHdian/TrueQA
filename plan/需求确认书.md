# 真心话抽卡对战小程序 · 完整需求规格 v1（已逐项确认）

> 经 9 轮需求确认，以下为最终结论。开工后如需微调，以本文件 + 聊天记录为准。

## 0. 技术架构
- **前端**：微信小程序原生（WXML/WXSS/JS）。主包 < 2MB；对战相关页面走 `subpackages/`。
- **后端**：自建 Node（你有云服务器）。Express 提供 REST；`ws` 提供 WebSocket 实时对战同步。
- **联调**：先用开发者工具「不校验合法域名」本地联调；真机前再配 HTTPS 域名白名单 + MP 后台。
- **AI 评分**：首版**留接口、怒气值手填**；后续接大模型（混元/OpenAI/规则，待定）。
- **美术**：代码绘制像素风（Canvas 2D / CSS）。全局**中性灰白**，**仅对战页**用西部沙漠背景。
- **合规**：首次启动隐私协议弹窗（`wx.requirePrivacyAuthorize`）；用户生成内容预留 `msgSecCheck` 接入点。

## 1. 用户系统
- 登录：`wx.login` → code → Node 用 AppID/AppSecret 换 openid + 自建 session token（存 `access_token`）。
- 资料：头像昵称走最新规则（`button open-type="chooseAvatar"` + `input type="nickname"`）。
- 数据：`users(openid, nick, avatar, level, exp, hp_base)`。
- 等级：L1 血量 20，每级 +10；升级经验阈值 = 当前等级 × 100。

## 2. 题库
- 数据：`questions(id, content, anger 0-10, ai_reason, source[system/user], owner_openid)`。
- 系统题：10 道三观/价值观简答题（已拟，可改），作普通弹兜底。
- 手动添加：输入内容 + 怒气值（0-10 滑块，可改）。
- **批量导入**：粘贴 → 按行解析 → **逐题确认**（每题保留可编辑输入：题目文字 + 怒气值）→ 批量写入。
  - 可选格式 `题目|怒气值(0-10)`；未带值默认 5，预览里可改。
  - 对齐市面「粘贴→解析→预览→确认」范式（参考礼账批量录入）。
- 列表：展示题目 + 怒气值，可编辑 / 删除。

## 3. 单人抽卡（= 抽卡 tab = 首页）
- 上方单人抽卡区：**老虎机滚动动画**抽题 → **翻牌（rotateY 180°）**显示题目。
- 题目**优先用户自建题**；用户无题时回退系统题（保证永远能抽）。
- 翻牌后**仅展示题目供作答**，无分享 / 复制 / 换一张等额外操作。
- 下方「**邀请好友对战**」按钮 → 进入房间流程。
- 分享能力：`onShareAppMessage` / `onShareTimeline` 由胶囊转发（结果页不加自定义分享按钮）。

## 4. 西部对战（分包，二人对战）
- 入口：抽卡页「邀请好友对战」→ 创建房间 → `onShareAppMessage` 带 `roomId` → 对手点卡片进房。
- 入场状态：1 人 → 弹「邀请对手」；2 人 → 弹「是否开战」。
- **先手：邀请者（房主）优先**。
- 弹药：高级弹（用户题，伤害 = 怒气值）/ 普通弹（系统题，伤害 = 怒气值 × 0.8）。
  - **高级弹用尽自动转普通弹**；普通弹数量无限。
- **抽取弹药 = 老虎机滚动动画**：点「抽取弹药」→ 老虎机滚动题目片段 → 停下显示抽到的题 + 弹种 → **手动装填** → 开枪。
- 回合流程：抽取(老虎机) → 装填 → 开枪 → **子弹中途停滞展开题目** → 受击方**文本作答**（语音首版不做）→ 发射方选倍率（不满意 ×1.2 / 普通 ×1.0 / 较好 ×0.5 / 超好 ×0）→ 受击 / 阵亡。
- 像素牛仔对枪动画（装弹 / 开枪 / 受击 / 阵亡），Canvas 2D 逐帧。
- 血量：L1=20，每级 +10；血量变动带入场 / 飘字动画。
- 掉线：房主掉线房间解散并通知对手；普通掉线允许短时重连，超时判负。
- **结束即散房**（不保留房间，再来需重建/重分享）。
- 结算页（双方均看）：伤害值 + 回答评价 + **综合评级 S/A/B/C/D** → 经验 → 升级。
  - **评级公式（自定义权重）**：
    `score = 0.4×win + 0.3×answerQuality + 0.3×hpRatio`
    - `win`：胜=1 / 负=0
    - `answerQuality = 1 − clamp(对方对我选的倍率均值 / 1.2, 0, 1)`
    - `hpRatio`：结算时剩余血量 / 满血（败方=0）
    - 映射：≥0.85→S，≥0.70→A，≥0.50→B，≥0.30→C，其余→D
  - **经验**：S+100 / A+70 / B+50 / C+30 / D+10；升级阈值 = level × 100。

## 5. 页面 / 导航
- tabBar 三栏：**抽卡（首页）/ 题库 / 我的**。
- 对战页（房间、对战、结算）走分包，从抽卡页按钮进入。

## 6. 开发排期
- **P0 地基**：脚手架、request 层、登录、数据模型、系统题种子、环境配置。
- **P1 单人抽卡**：老虎机 + 翻牌，优先用户题。
- **P2 题库管理**：手动添加 + 批量导入（逐题确认）。
- **P3 西部对战**：房间 / 分享 / 实时同步 / 动画 / 结算。
- **P4 迭代**：语音作答、音效、排行榜、AI 评分接入、更多场景、3+ 人房间。

## 7. 实现期默认值（可改）
- WebSocket 用 `ws` 库；房间状态机 `waiting / fighting / finished`。
- 断线重连超时 ~30s；房间创建后未开战过期 ~5min。
- 题目长度上限、怒气值 0-10 校验。
- 个人页展示：等级、经验条、血量基数、我的题数、对战记录（本地）。
- 分享卡片封面：canvas 生成或固定图。
